## 2. 现阶段最可能原因（在上述冗余逻辑去除后）

当“可视化路径正确但机床加工范围偏移”仍存在时，最可能问题将集中在**机床运动学建模本体**，而不是路径规划：

1. **回转链变换顺序与真实机床结构不一致**  
   当前逆解采用固定链：`inv(C * (T * A))`。若真实机床是另一种几何链（如 `inv(C * A * T)` 或 `inv(T * C * A)`），会出现整体偏移或随姿态变化偏移。

2. **pivotOffset 符号/方向与机床定义不一致**  
   `pivotOffsetY/Z` 的方向约定若与机床控制器定义相反，会导致稳定偏置，尤其在 A/C 变化后偏差更明显。

3. **刀轴方向约定不一致（刀具朝向 +Z/-Z）**  
   当前 A/C 解算与 `toolAxis` 的符号约定固定；若 CAM 输出刀轴定义与机床控制器期望相反，会导致角度“看似合理但空间位置错误”。

4. **角度限幅引发的姿态失真**  
   当真实目标姿态超出 A 轴限制时，当前逻辑会截断角度，可能造成局部位置偏差放大（尤其 5 轴侧壁/底部姿态）。

## 3. 建议修改方案（不贴完整代码）

### 方案 A：先做“运动学闭环验证”

在后处理增加一个调试开关，对每个点执行：

- 逆解：`tip/toolAxis -> X/Y/Z/A/C`
- 正解：`X/Y/Z/A/C -> tip'`
- 记录误差：`||tip' - tip||`

若误差随姿态系统性变化，可快速确认是链路建模问题（而非 G55、非路径规划）。

### 方案 B：将回转链做成显式机型模板

为 `xyzac-trt` 增加可配置的链顺序（例如 2~3 个明确模板），而不是写死一条链：

- 模板 1：`C * (T * A)`（当前）
- 模板 2：`C * (A * T)`
- 模板 3：`T * (C * A)`

通过基准试切数据选定模板。

### 方案 C：增加姿态约束与告警

当角度被限幅时，输出告警并记录点号/目标姿态/限幅后姿态，避免“静默失真”。

### 方案 D：输出程序级诊断信息

在 G-code 头部注释增加：

- CL 点包围盒
- 机床轴包围盒
- 限幅次数统计
- 当前机型模板标识

便于现场一眼定位是“几何链问题”还是“输入路径问题”。

## 4. 现场快速验证步骤

1. 固定同一工件、同一对刀、同一 G55。  
2. 取 3 个已知基准点（A0C0 / A90C0 / A0C90）做单点程序。  
3. 比较“理论刀尖位置 vs 实测触点”。  
4. 若误差在不同姿态下方向/量级变化明显，优先排查回转链顺序与 pivotOffset 定义。  
5. 若误差近似常量，再排查工装/对刀/刀长补等现场因素。

在已彻底去除 `workOffset` 与 `rtcp` 冗余路径后，当前问题最应优先聚焦到 **逆解运动学链与机床实际几何定义是否一致**。下一步最有效的是先补“逆解-正解闭环误差诊断”，再根据数据调整回转链模板与 pivot 参数约定。
